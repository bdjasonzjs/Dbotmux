# 新手引导 ·「加成员」步 UX 改造验收（松松实测反馈）

> 背景：松松实测「加成员」步说"连我都不知道填啥"——空白行 + 没解释 + 出现"模型"栏（违反"配模板不碰 bot"）。
> 范围：只改 `task-team-onboarding.ts` 的「加成员」步（+少量 css）。复用现有 canvas-data，停 working tree、未 commit/部署/碰编译配置。

## 5 条改造（松松点名）逐条落地

1. **默认就摆好一套填好的示例**：一进「加成员」默认已有三个角色、带有意义示例名——**开发工程师**（干活的）/ **代码审查员**（把关的）/ **进度观察员**（盯梢的）。照着改名/增删即可，不是空白。（mount 时 `addRole('developer'/'reviewer'/'observer')`，默认名取 `KIND_SAMPLE_NAME`。）
2. **删掉「模型」栏**：每行的"模型(可空…)"输入框已移除；node.model 不在配模板这步产生。模型/引擎归第二段「建真群·挑 bot」（挑了哪个 bot 自带引擎；盯梢的默认便宜 bot 在那步体现）。
3. **字段说人话 + 给例子**：名字框 placeholder = "给这个角色起个名，比如'前端工程师'"；每个角色类型带一句副标题：干活的=负责执行出活 / 把关的=负责审把关质量 / 汇报的=负责对外汇报 / 盯梢的=只看进展不插手（也作 add 按钮 tooltip）。
4. **右侧预览有意义**：节点上行=角色名（开发工程师）、下行=干啥（干活的），不再"干活的/干活的"重复。
5. **顶部一句话引导**：标题"给这个小组配上角色：谁干活、谁审、谁盯" + "下面已经给你摆了一套常见的，改改名就行；这步只配角色，不挑机器人——机器人在下一段「建真群」再选"。
